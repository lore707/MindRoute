```
  - "Mapping" pill — shows trait chips being mapped (small rounded pills)
[Action buttons]     — Back (left, ghost style) + Continue (right, navy solid, rounded)
```

### Continue Button Behavior (CRITICAL — must work)
- The Continue button starts **disabled** (opacity 0.3, pointer-events none)
- When the user types MORE THAN 10 characters in the textarea, the button becomes **enabled** (opacity 1, pointer-events all, navy background)
- Clicking Continue when enabled advances to the next slide
- Pressing Enter (without Shift) when the button is enabled also advances
- Pressing Shift+Enter creates a new line in the textarea
- Use EVENT DELEGATION on the parent container, not individual event listeners per button

### Back Button Behavior
- Hidden on Q1 (first question has no back)
- On all other questions, clicking Back returns to the previous slide
- The previous textarea value is preserved when going back

### Slide Transitions
- When advancing: current slide fades out, new slide animates in with slideUp (translateY 28px → 0, opacity 0 → 1, 0.55s ease)
- Use `display: none` / `display: block` toggling with an `.active` class

## TECHNICAL REQUIREMENTS

### JavaScript Architecture
Use EVENT DELEGATION — attach event listeners to a single parent container, not to individual buttons. This prevents bugs when slides are hidden/shown.

```javascript
// Example pattern:
document.getElementById('quizContainer').addEventListener('click', function(e) {
  const nextBtn = e.target.closest('.btn-next');
  const backBtn = e.target.closest('.btn-back');
  
  if (nextBtn && nextBtn.classList.contains('enabled')) {
    const targetSlide = parseInt(nextBtn.getAttribute('data-to'));
    goToSlide(targetSlide);
  }
  
  if (backBtn) {
    const targetSlide = parseInt(backBtn.getAttribute('data-to'));
    goToSlide(targetSlide);
  }
});
```

### State Management
- Store all textarea answers in a JavaScript object: `answers = {}`
- When navigating away from a slide, save the textarea value
- When navigating back to a slide, restore the textarea value
- Auto-focus the textarea when a slide becomes active (with 100ms delay)

### Slide Visibility
- Use `display: none` by default for all slides
- Add `display: block` + `.active` class to show a slide
- Only ONE slide should be visible at any time

### CSS Animations
- `slideIn`: translateY(28px) → translateY(0), opacity 0 → 1, 0.55s cubic-bezier(0.16, 1, 0.3, 1)
- `drift`: slow floating motion for background orbs, 20-30s duration
- `pulse`: opacity pulsing for current progress segment
- `twink`: fade in/out for decorative micro-particles
- `spin`: 360deg rotation for orbit rings, 45-60s duration

### Textarea Behavior
- Min height: 160-170px
- Max length: 800 characters
- Border: 1.5px solid #E4E0DA
- Focus state: coral-light border + coral glow shadow
- Rounded corners: 18-20px
